# Study AI

Upload a document and get a concise **summary** plus an interactive **multiple-choice quiz**, generated by the OpenAI API. Supports PDF, DOCX, PPTX, TXT/Markdown, and images (via OCR).

## Features

- Drag-and-drop upload (PDF · DOCX · PPTX · TXT · PNG/JPG and other images)
- Text extraction: `unpdf` for PDF, `mammoth` for DOCX, `officeparser` for PPTX, native read for text, `tesseract.js` OCR for images
- AI summary + MCQs in one structured call (OpenAI **Structured Outputs**, strict JSON schema)
- Choose how many questions (1–20, **default 5**); every question has 4 options with the correct answer marked
- Interactive quiz with instant right/wrong feedback, explanations, and a running score
- Extracted-text preview, loading states, and clear error messages
- Graceful handling of unsupported files, empty/scanned documents, and API failures

## Requirements

- **Node.js 18.17+** (developed on Node 22)
- An **OpenAI API key** with access to the chosen model

## Setup

```bash
# 1. Install dependencies
npm install

# 2. Add your API key
cp .env.example .env
#    then edit .env and set OPENAI_API_KEY=sk-...

# 3. Start the server
npm start
```

Open **http://localhost:3000**.

> `npm run dev` starts the server with file-watching for development.

## Configuration

All optional, set in `.env`:

| Variable          | Default        | Description                                              |
| ----------------- | -------------- | -------------------------------------------------------- |
| `OPENAI_API_KEY`  | _(required)_   | Your OpenAI API key.                                     |
| `OPENAI_MODEL`    | `gpt-4.1-mini` | Any model that supports Structured Outputs.              |
| `PORT`            | `3000`         | Server port.                                             |
| `MAX_FILE_MB`     | `15`           | Maximum upload size in MB.                               |
| `MAX_TEXT_CHARS`  | `24000`        | Max characters of extracted text sent to the model.      |

## How it works

```
Browser ──upload──▶ Express (/api/analyze)
                      │
                      ├─ extract text  (unpdf | mammoth | fs | tesseract.js)
                      ├─ validate      (type, size, non-empty)
                      ├─ generate      (OpenAI chat.completions + json_schema)
                      └─ return JSON ──▶ summary + questions + preview
```

The key never leaves the server — the browser only talks to your own `/api/analyze` endpoint.

## Notes

- **Image OCR** downloads a language model (`eng`) and the Tesseract core on first use, so the first image can take longer and needs internet access.
- **Scanned PDFs** with no text layer can't be read by `pdf-parse`. Export the page as an image to use OCR instead.
- **Legacy `.doc`** isn't supported — save as `.docx`, PDF, or TXT.
- Long documents are truncated to `MAX_TEXT_CHARS` before being sent to the model; the UI flags when this happens.
